## Part A: ML Retraining Moved to Admin Portal

### API Service (v0.26.0)

**New endpoint:** `POST /api/v1/admin/system/ml/retrain`
- Requires `platform_admin` role via `require_admin_role("platform_admin")`
- Request: `{ "force": false, "min_samples": 200 }`
- Response: `{ "success": true, "samples_used": 250, "accuracy": 0.87, "auc": 0.91, "message": "..." }`
- Audit logged as `admin.ml.retrain` (success and failure)

**Legacy endpoint deprecated:** `POST /api/v1/ml/retrain`
- Now requires `platform_admin` role (returns 403 for non-admins)
- Marked `deprecated=True` in OpenAPI spec

**Files changed:**
- `src/presentation/api/v1/endpoints/admin/system.py` ‚Äî Added `MLRetrainRequest`, `MLRetrainResponse`, `admin_retrain_model()`
- `src/presentation/api/v1/endpoints/ml.py` ‚Äî Added admin role check, deprecated decorator

### Admin Portal (v0.2.0)

**New page:** ML Models (`/ml-models`)
- Model Status section (trained/untrained, accuracy, AUC, version)
- Training Data Stats (sample counts by source, class distribution)
- Scanner Quality Table (per-scanner precision metrics)
- Retrain Model button (standard: 200+ samples)
- Force Retrain button (minimum: 50 samples)

**Files changed:**
- `src/pages/AdminMLModels.tsx` ‚Äî New page
- `src/lib/api/admin.ts` ‚Äî Added `retrainModel()`, `RetrainModelRequest/Response`
- `src/App.tsx` ‚Äî Added `/ml-models` route
- `src/layouts/AdminLayout.tsx` ‚Äî Added nav item with `CpuChipIcon`

### Dashboard (v0.39.0)

**Retrain functionality removed:**
- `ScannerQualityPage.tsx` ‚Äî Removed retrain mutation, button, error display
- `ModelStatusWidget.tsx` ‚Äî Made read-only (removed `showRetrain` prop, retrain button)
- `ScannerQualityTable.tsx` ‚Äî Improved empty-state message explaining labeling requirements

## Part B: Scanner Quality Empty-State UX

The "No scanner quality data available yet" message now explains:
1. Quality metrics are generated by labeling vulnerabilities as Confirmed or False Positive
2. Labels are applied from the vulnerability detail page
3. Each scanner needs at least 10 labeled findings before metrics appear

No backend changes ‚Äî UX copy improvement only.
