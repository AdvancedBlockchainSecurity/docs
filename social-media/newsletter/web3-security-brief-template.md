## Subject Line Formula Bank

**Pattern 1: Urgency + Stat**
- "[X]% of contracts have this vulnerability—here's the fix"
- "$[Amount] lost to [attack type]—are you protected?"
- "Critical: [Vulnerability] found in [X]% of scans"

**Pattern 2: Curiosity + Value**
- "The security flaw hiding in your initialize() function"
- "What the $[Amount] hack teaches us about [topic]"
- "The overlooked vulnerability in [common pattern]"

**Pattern 3: News Hook**
- "[Protocol] exploit: Technical breakdown + prevention"
- "New OWASP Top 10 reveals surprising #1 risk"
- "[Tool/Standard] released—what it means for your security"

**Pattern 4: How-To**
- "How to prevent [vulnerability] in 3 steps"
- "Building a multi-layer security stack"
- "Securing [specific pattern] the right way"

**Pattern 5: Insight/Analysis**
- "Why 80% of hacks aren't code vulnerabilities"
- "The real cost of skipping security audits"
- "What [X] successful exploits have in common"

## Crisis/Exception Protocols

### Zero-Day Discovered
If a critical zero-day vulnerability is discovered between scheduled issues:
- Send emergency alert within 24 hours
- Keep to 300 words max
- Focus on: What it is, who's affected, immediate actions
- Link to detailed blog post
- Note that regular issue still coming on schedule

### Major Hack/Incident
If a significant exploit occurs just before send:
- Consider adjusting content to cover it (if time permits)
- If too late, address in next issue
- Post immediate Twitter/LinkedIn thread in meantime
- Newsletter remains authoritative deep-dive, not breaking news

### Content Error Discovered Post-Send
- Assess severity (minor typo vs. technical inaccuracy)
- If critical: Send correction to full list within 6 hours
- If minor: Note in next issue's opening
- Update web version immediately
- Document in lessons learned log

# Writing Tips for Newsletter Contributors

1. **Lead with impact**: First sentence should make readers care
2. **Show, don't just tell**: Use code examples, not just descriptions
3. **Write for skimmers**: Use bullets, bold, clear headers
4. **Vary sentence length**: Mix short punchy sentences with longer explanatory ones
5. **Active voice**: "Attackers exploited" not "The vulnerability was exploited"
6. **Numbers grab attention**: "$53M lost" beats "significant losses"
7. **Be specific**: "47% of contracts" beats "many contracts"
8. **Explain acronyms**: First use: "Multi-sig (multi-signature)"
9. **Link strategically**: 5-8 educational links per issue maximum
10. **End sections strong**: Last sentence should reinforce key takeaway
11. **Stay objective**: Present facts and analysis without promotional agenda
12. **Cite sources**: Link to original research, tools, and documentation
13. **Educational first**: Every section should teach something actionable
14. **Independent voice**: Be fair in tool comparisons, acknowledge trade-offs
