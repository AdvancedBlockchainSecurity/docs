## Detailed Fix Descriptions

### BSO-SEC-006/008: Redis Nonce Storage

**Files:**
- NEW: `src/infrastructure/auth/nonce_storage.py`
- MODIFIED: `src/presentation/api/v1/endpoints/wallet_auth.py`
- MODIFIED: `src/presentation/api/v1/endpoints/solana_wallet_auth.py`

**Changes:**
- Created Redis-backed nonce storage with automatic TTL
- Uses SETEX for atomic store-with-expiration
- Uses GETDEL for atomic check-and-delete (prevents TOCTOU)
- Fallback to in-memory for local development

### BSO-SEC-007: Session Token Response Body

**File:** `src/presentation/api/v1/endpoints/admin/auth.py`

**Changes:**
- Removed `session_token` from JSON response body
- Session token now only available via:
  - `X-Admin-Session` response header (for API/CLI)
  - `admin_session` httpOnly cookie (for browser)

### BSO-SEC-009: JWT Token TTL

**File:** `src/infrastructure/config.py`

**Changes:**
- Reduced `jwt_access_token_expire_minutes` from 180 to 60 minutes
- Added comment referencing security best practices
